## cuco.plug.v3（MIOT 插座/灯）

**本型号设备**：乌龟灯、走廊灯。

走 MIOT 协议，用 `miiocli miotdevice`，开关属性为 **siid=2, piid=1**。参数按 Python 字面量传，布尔用 **True/False（大写）**。

### 乌龟灯（已验证）

- IP: `192.168.50.220`  
- token: `e6c9cc7e3041298eb0bd32ad75f11d33`

**开灯（已验证）：**

```bash
miiocli miotdevice --ip 192.168.50.220 --token e6c9cc7e3041298eb0bd32ad75f11d33 raw_command set_properties '[{"siid":2,"piid":1,"value":True}]'
```

**关灯（已验证）：**

```bash
miiocli miotdevice --ip 192.168.50.220 --token e6c9cc7e3041298eb0bd32ad75f11d33 raw_command set_properties '[{"siid":2,"piid":1,"value":False}]'
```

### 定时/延时（cuco.plug.v3）

- 官方 API 与发现步骤见 [延时与定时API说明.md](延时与定时API说明.md)。
- **已明确**：**piid=3** = 延时时长（写 5 时米家 App 显示 5 分钟）；**piid=1** = 延时开启开关（False=关，True=开）。仅设 piid=3 只会改 App 里的「延迟时间」，不会触发延时，需**同时把 piid=1 设为 True** 才会执行「延时开启」。

**验证情况**：  
- 设 piid=1=False、piid=3=2 均返回 code 0，但 App 的「延时开启」「延迟时间」均无变化；  
- **全网搜索 + 抓包验证**后结论：  
  - **功能实现方式**：米家 App 的「延时开启/关闭」**不是直接调用设备的 miot 属性**，而是通过**米家云端 API 创建定时场景**（`/app/scene/edit`）实现的。场景由云端计时，到时下发开/关命令给设备。  
  - **本地 API 调用无效原因**：设备的 siid=4 属性（延时相关）无法通过本地 `set_properties` 触发倒计时逻辑，只能由云端场景驱动。  
  - **替代方案**：用外部脚本 `time.sleep()` + 发开/关命令实现延时（简单、本地、不依赖云端）。  
- **当前已验证可用**：仅**开关（siid=2, piid=1）**。详见 [延时与定时API说明.md](延时与定时API说明.md) 第八节"抓包验证结论"。

#### 本地延时实现（hzr_mcp 集成）

已在 `hzr_mcp.py` 中实现外部脚本延时方案，支持语音控制：

**乌龟灯延时功能**：
- `wuguideng_delayed_on(delay, unit)` - 延时开启（支持秒/分钟/小时）
- `wuguideng_delayed_off(delay, unit)` - 延时关闭（支持秒/分钟/小时）
- `wuguideng_cancel_delay()` - 取消延时任务

**语音示例**：
- 「30秒后开启乌龟灯」→ 30秒后自动开启
- 「5分钟后关闭乌龟灯」→ 5分钟后自动关闭
- 「1小时后关乌龟灯」→ 1小时后自动关闭
- 「取消乌龟灯延时」→ 取消所有延时任务

**特性**：
- ✅ 新指令自动覆盖旧的延时任务
- ✅ 本地计时，不依赖云端和网络
- ✅ 支持多设备独立延时（乌龟灯、走廊灯互不影响）

## yodq.humidifier.yo001

**本型号设备**：YO Humidifier（YO 加湿器）。

### 设备信息
- IP: `192.168.50.203`  
- Token: `f7f296300d02ac9b5bf0b13d1a3c7995`
- 硬件: ESP32C3
- 固件: 1.0.6

### 已验证的控制命令

**开关控制**：

```bash
# 打开加湿器
miiocli miotdevice --ip 192.168.50.203 --token f7f296300d02ac9b5bf0b13d1a3c7995 raw_command set_properties '[{"siid":2,"piid":1,"value":True}]'

# 关闭加湿器
miiocli miotdevice --ip 192.168.50.203 --token f7f296300d02ac9b5bf0b13d1a3c7995 raw_command set_properties '[{"siid":2,"piid":1,"value":False}]'

# 查询开关状态
miiocli miotdevice --ip 192.168.50.203 --token f7f296300d02ac9b5bf0b13d1a3c7995 raw_command get_properties '[{"siid":2,"piid":1}]'
```

### 已发现的属性

| siid | piid | 功能 | 类型 | 当前值 | 状态 |
|------|------|------|------|--------|------|
| 2 | 1 | 电源开关 | 布尔 | False | ✅ 已验证 |
| 2 | 2 | 模式/档位 | 整数 | 0 | ⚠️ 待测试 |
| 2 | 9 | 未知参数A | 整数 | 2 | ❓ 待测试 |
| 2 | 10 | 未知参数B | 整数 | 1 | ❓ 待测试 |
| 4 | 1 | 辅助开关 | 布尔 | False | ⚠️ 待测试（可能是LED/蜂鸣器）|
| 5 | 1 | 数值参数 | 整数 | 25 | ⚠️ 待测试（可能是目标湿度）|

**注意**：
- siid=3（传感器数据）不支持（code=-4003）
- siid=2 的 piid=3-8 不支持
- python-miio 库尚未官方支持此型号，需使用 `raw_command`

### 本地延时实现（hzr_mcp 集成）

已在 `hzr_mcp.py` 中实现，支持语音控制：

- `jiashiqi_delayed_on(delay, unit)` - 延时开启
- `jiashiqi_delayed_off(delay, unit)` - 延时关闭
- `jiashiqi_cancel_delay()` - 取消延时任务

**语音示例**：
