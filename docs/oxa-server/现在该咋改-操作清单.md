## 自建小智 server：让验证码能播报

**现象**：连官网小智时，未绑定会语音播报验证码；连自建时，日志有验证码但音箱不播。

**原因**：Open-XiaoAI 只有收到 `{"type":"tts","state":"start"}` 才会进入 SPEAKING 并播放后续音频；自建 server 可能没发这条或顺序错了。

**改法**（在 xiaozhi-esp32-server 源码里做）：

1. 找到向 WebSocket 客户端发 TTS 的代码（NAS 日志里出现 `[core.handle.sendAudioHandle]` 的那个模块，一般在 **`core/handle/sendAudioHandle.py`** 或 TTS 输出到 WebSocket 的等价位置）。
2. 在**发任何一段 TTS 二进制音频之前**，先发一条 JSON：
   ```json
   {"type": "tts", "state": "start", "session_id": "..."}
   ```
   （`session_id` 可与当前连接一致或留空。）
3. 发完所有该段 TTS 的音频后，再发：
   ```json
   {"type": "tts", "state": "stop", "session_id": "..."}
   ```
4. 顺序务必是：**start → 音频帧 → stop**。不能先发音频再发 start。

若你用的是 xinnan-tech/xiaozhi-esp32-server，需要从该仓库或 NAS 上的代码里找到上述发送逻辑并按上面顺序加/改。

**验证**：改完后未绑定设备时唤醒，听音箱是否播报验证码。
