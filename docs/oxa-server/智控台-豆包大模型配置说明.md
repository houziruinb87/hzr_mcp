# 智控台「豆包大模型」配置说明（火山方舟）

## 一、当前情况对比

| 项目 | 你开通的（火山后台） | 智控台当前填写 | 是否一致 |
|------|----------------------|----------------|----------|
| **服务/模型** | **Doubao-Seed-2.0-Code**（豆包 Seed 2.0 代码模型） | **doubao-1-5-pro-32k-250115**（豆包 1.5 Pro） | ❌ **不一致** |
| **Base URL** | 对话 API：`https://ark.cn-beijing.volces.com/api/v3/chat/completions` | `https://ark.cn-beijing.volces.com/api/v3` | ✅ 正确 |

结论：
- **基础 URL** 是对的，智控台里填 `https://ark.cn-beijing.volces.com/api/v3` 即可（小智会按 OpenAI 兼容方式拼出 `/chat/completions`）。
- **模型名称** 不对：你开通的是 **Doubao-Seed-2.0-Code**，智控台里却填的是 **doubao-1-5-pro-32k-250115**（豆包 1.5 Pro），是另一个模型，调用会走错服务或鉴权失败。

---

## 二、Doubao-Seed-2.0-Code 应填的模型名称

根据火山方舟文档与 API 说明：

- 使用 **API Key 鉴权** 时，可直接使用**模型名称**（预置推理接入点），无需先创建 `ep-xxx` 接入点。
- **Doubao-Seed-2.0-Code** 在 Chat API 中使用的 **model 参数** 为：**`doubao-seed-2.0-code`**（小写、连字符）。

因此，智控台里「豆包大模型」应填：

- **模型名称**：`doubao-seed-2.0-code`
- **基础 URL**：`https://ark.cn-beijing.volces.com/api/v3`（保持即可）

---

## 三、智控台修改步骤

1. 智控台 → **模型配置** → **大语言模型** → 找到「豆包大模型」(DoubaoLLM) → **修改**。
2. **调用信息** 中：
   - **基础 URL**：`https://ark.cn-beijing.volces.com/api/v3`（不变）。
   - **模型名称**：由 `doubao-1-5-pro-32k-250115` 改为 **`doubao-seed-2.0-code`**。
3. **API 密钥** 保持你在火山后台复制的 Key 不变。
4. 点击 **保存**。

保存后，小智会使用你已开通的 **Doubao-Seed-2.0-Code** 进行对话。

---

## 四、调通测试结果（当前）

用你 NAS 上的 API Key 和 Base URL 做了请求测试：

| 请求 model | 结果 |
|------------|------|
| `doubao-seed-2.0-code` | **404** `InvalidEndpointOrModel.NotFound`（该 model/接入点不存在或当前无权限） |
| `doubao-1-5-pro-32k-250115` | **404** `ModelNotOpen`（账号未开通该模型） |

说明：
- **Base URL** 和 **API Key** 有效（能连上并返回明确错误）。
- 仅「开通」Doubao-Seed-2.0-Code 还不够，需在火山方舟 **创建推理接入点**，并把接口里用的 **接入点 ID（ep-xxx）** 当作「模型名称」填到智控台。

**建议操作：**
1. 登录 [火山方舟控制台](https://console.volcengine.com/ark/region:ark+cn-beijing) → **模型调用** → **推理接入点**（或「在线推理」）。
2. **创建推理接入点**，选择已开通的 **Doubao-Seed-2.0-Code**，创建成功后得到 **接入点 ID**（形如 `ep-20241201xxxxxx-xxxx`）。
3. 智控台「豆包大模型」里：**模型名称** 填该 **接入点 ID**（如 `ep-20241201xxxxxx-xxxx`），Base URL 和 API Key 不变，保存后再测。
