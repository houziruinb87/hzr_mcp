# 唤醒后对话无反应（自建 xiaozhi-esp32-server）

## 现象

- 说「小智同学」能听到「你好主人，我是小智，请问有什么吩咐？」
- 再说一句（如「今天天气怎么样」）没有反应，也听不到验证码提示。

## 原因简述

1. **OTA 地址错误**：Open-XiaoAI 的 `OTA_URL` 若指向主服务 `xiaozhi-esp32-server:8000`，在全模块部署下 OTA POST 可能被关闭或无法正确返回设备激活信息。
2. **设备未在智控台绑定**：自建小智要求设备在智控台用**验证码**绑定后，WebSocket 对话才会正常（否则可能连不上或连上后被断开）。

## 已做修改（config.py 里 xiaozhi 两处地址）

**要点**：OTA 必须指向**智控台 web**（端口 8002/18002），不能指向主服务 8000；对话 WebSocket 指向**主服务**（8000）。

按 Open-XiaoAI 是否与 xiaozhi 在同一 Docker 网络，二选一：

- **与 xiaozhi 同 Docker 网络**（同一 compose 或加入 xiaozhi 网络）时，可用容器名：
  - `OTA_URL`: `http://xiaozhi-esp32-server-web:8002/xiaozhi/ota/`
  - `WEBSOCKET_URL`: `ws://xiaozhi-esp32-server:8000/xiaozhi/v1/`
- **Open-XiaoAI 单独一个 compose**（当前 NAS 常见情况）时，用 **NAS 宿主机 IP**：
  - `OTA_URL`: `http://192.168.1.100:18002/xiaozhi/ota/`
  - `WEBSOCKET_URL`: `ws://192.168.1.100:8000/xiaozhi/v1/`

（NAS IP 换成你的实际地址。）这样 Open-XiaoAI 能正确拿到 OTA 激活信息（含验证码），并与小智主服务建立 WebSocket 对话。

## 你需要做的：在智控台绑定设备

1. **打开智控台**  
   浏览器访问：**http://192.168.1.100:18002**（NAS 的 IP 换成你的）。

2. **登录**  
   若未注册，先注册并登录。

3. **设备管理 → 添加设备**  
   - 在菜单里找到「**设备管理**」或「**添加设备**」。
   - 当前 Open-XiaoAI 使用的设备 ID 为容器内 MAC（如 `02:42:c0:a8:50:02`），首次 OTA 成功后会生成 **6 位验证码**。

4. **输入验证码**  
   - 验证码会在 **Open-XiaoAI 日志**里出现（小智通过 TTS 下发的验证码会在终端打印为「🤖 小智：…」）。
   - 当前 Open-XiaoAI 设备（如 `02:42:c0:a8:50:02`）向智控台 OTA 请求时，服务端会返回一次性的 **6 位验证码**（例如 `509441`），可在智控台「添加设备」处输入该验证码完成绑定（若已过期可重启 Open-XiaoAI 再触发一次 OTA 获取新码）。
   - 若尚未听到或看到，可先到 NAS 上执行一次重启并立刻查看日志：
     ```bash
     docker restart open-xiaoai-xiaozhi
     docker logs -f open-xiaoai-xiaozhi
     ```
     然后对音箱说「小智同学」再随便说一句话，看日志里是否出现「验证码」或「控制面板」及数字。
   - 在智控台「添加设备」处输入该 **6 位验证码**，完成绑定。

5. **再次对话测试**  
   绑定成功后，再说「小智同学」→「今天天气怎么样」应能收到小智回复。

## 关键日志证据说明（服务在运行、有连接、有验证码）

某次在 NAS 上执行 `docker logs open-xiaoai-xiaozhi --tail 50` 时，出现过下面这类输出，说明**整条链路曾一度打通**：

| 日志行 | 含义 |
|--------|------|
| `✅ 已启动: "0.0.0.0:4399"` | Open-XiaoAI 已在本机 4399 端口监听，服务在运行。 |
| `✅ 已连接: 192.168.96.1:59512` | 有客户端（如音箱或手机端）连上了 Open-XiaoAI，来源 IP 为 192.168.96.1，源端口 59512。 |
| `💬 我说：请登录控制面板，输入075837，绑定设备` | 自建小智返回了「需要绑定设备」的文案，并下发了 **6 位验证码 075837**（“我说”表示上行/请求侧看到的回复内容）。 |
| `🤖 小智：请登录控制面板，输入075837，绑定设备。` | 小智通过 TTS 播报的同一句提示（含验证码 075837），说明 Open-XiaoAI 已连上自建小智并收到了语音回复。 |
| `👋 已退出唤醒` | 本次唤醒会话结束。 |

**由此可以得出的结论：**

1. **Open-XiaoAI 在运行**：已成功监听 0.0.0.0:4399。
2. **有设备连上 Open-XiaoAI**：192.168.96.1 与 Open-XiaoAI 建立了连接（可能是音箱、同一局域网内的手机或其它客户端）。
3. **Open-XiaoAI 已连上自建小智**：能收到小智的「绑定设备」提示和验证码，说明 OTA/WebSocket 到自建小智（如 `http://NAS:18002/xiaozhi/ota/`、`ws://NAS:8000/xiaozhi/v1/`）是通的。
4. **验证码已下发**：该次会话的 **6 位验证码为 075837**。若当时未在智控台绑定，可在此类日志中查找「输入xxxxxx」里的数字，或再次触发一次对话/重启 Open-XiaoAI 获取新验证码后在智控台添加设备。

若之后出现「无法唤醒」或「无反应」，而之前曾出现过上述日志，则多半是：**客户端（如音箱）未连到当前这台 Open-XiaoAI**（例如未配置到 NAS IP:4399、或指到了别的机器）。可先恢复为「仅用宿主机 IP + 单网络」的最简配置，并确认音箱/客户端指向 NAS:4399，再试唤醒。

## 若仍无反应

- 确认 Open-XiaoAI 与 xiaozhi 在同一 Docker 网络（如 `xiaozhi_server_default`），能解析 `xiaozhi-esp32-server` 和 `xiaozhi-esp32-server-web`。
- 在 NAS 上同时看两个日志，边说话边观察：
  ```bash
  docker logs -f open-xiaoai-xiaozhi
  docker logs -f xiaozhi-esp32-server
  ```
  看是否有连接错误、超时或 403 等。
- 智控台「参数管理」里确认 **server.ota**、**server.websocket** 与当前 NAS 地址一致（如 `http://192.168.1.100:18002/xiaozhi/ota/`、`ws://192.168.1.100:8000/xiaozhi/v1/`）。
