# 小智 server 详细日志说明与问题定位

为便于排查「唤醒后追问无反应」等问题，在自建小智 server 中增加了带 `[小智]` 前缀的 INFO 级日志，并通过挂载补丁文件持久生效。

> **Open-XiaoAI 端**：当前使用官方镜像，无内置「详细日志」开关；如何看其日志、为何主要靠小智日志定位，见 [Open-XiaoAI日志与排查说明.md](./Open-XiaoAI日志与排查说明.md)。

## 一、补丁文件与挂载

- **位置（NAS）**：`xiaozhi_server/patches/`
  - `patches/core/connection.py`：WebSocket 连接与消息处理
  - `patches/core/api/ota_handler.py`：OTA 接口
- **挂载**：已在 `docker-compose.yml` 中配置，容器内会使用上述补丁覆盖镜像内同名文件，重启容器后仍生效。

## 二、新增日志含义

### 1. OTA 接口（设备/Open-XiaoAI 拉取配置时）

| 日志内容 | 含义 |
|----------|------|
| `[小智] OTA GET 请求到达 from=...` | 有人访问了 OTA 的 GET 接口（例如浏览器或健康检查）。 |
| `[小智] OTA POST 请求到达 from=... device-id=... client-id=...` | 设备或 Open-XiaoAI 向 OTA 提交了 POST（拉取服务器配置、WebSocket 地址等）。**若追问前能看到这条，说明至少 OTA 被正确调用了。** |

**如何看**：  
`docker logs xiaozhi-esp32-server --tail 200 2>&1 | grep 小智`

### 2. WebSocket 连接（真实对话走这里）

| 日志内容 | 含义 |
|----------|------|
| `[小智] ========== 新 WebSocket 连接 ==========` | 有客户端连上了小智的 WebSocket（8000 端口）。 |
| `[小智] 连接详情 IP=... device-id=... path=...` | 连接来源 IP、设备 ID、请求 path，用于确认是否是 Open-XiaoAI。 |
| `[小智] 收到文本消息 len=... 内容前100字: ...` | 收到文本（如 hello、或用户说的「北京天气如何」）。**有这条说明连接已建立且消息到达小智。** |
| `[小智] 收到音频消息 len=... bytes` | 收到音频数据（语音识别前会先收到音频）。 |
| `[小智] 客户端断开连接 IP=... device-id=...` | 该 WebSocket 连接被关闭。 |

**若追问时一条 WebSocket 相关日志都没有**：说明 Open-XiaoAI 没有连上自建小智的 WebSocket，或连的是别的地址/端口，需检查 Open-XiaoAI 的 `config.py` 里 `xiaozhi.WEBSOCKET_URL` 是否指向 `ws://NAS_IP:8000/xiaozhi/v1/`。

## 三、根据日志快速定位

### 现象：唤醒成功，追问「北京天气如何」没反应

1. **先看是否有 OTA POST**  
   - 有 `[小智] OTA POST 请求到达`：说明 Open-XiaoAI 在向自建小智拉配置，至少 OTA 地址是对的。  
   - 没有：可能是 Open-XiaoAI 仍在使用缓存的官方地址，或 `OTA_URL` 未指向自建（如 `http://NAS:18002/xiaozhi/ota/`）。

2. **再看是否有 WebSocket 连接**  
   - 有 `[小智] ========== 新 WebSocket 连接 ==========` 和 `[小智] 连接详情`：说明对话已走到自建小智，问题可能在设备绑定、LLM/TTS 或业务逻辑。  
   - 没有：说明**追问时 Open-XiaoAI 没有连上自建小智的 WebSocket**。要检查：  
     - `config.py` 里 `WEBSOCKET_URL` 是否为 `ws://NAS_IP:8000/xiaozhi/v1/`  
     - Open-XiaoAI 容器内能否访问 `NAS_IP:8000`（网络/防火墙）  
     - 智控台里设备是否已绑定（验证码绑定）。

3. **有连接、有文本消息，但无回复**  
   - 有 `[小智] 收到文本消息 ... 北京天气...`：说明请求已到小智。  
   - 再查：智控台设备绑定、LLM/ASR/TTS 配置、以及同一时间段小智 server 的 ERROR 日志。

## 四、查看日志命令（NAS 上执行）

```bash
# 最近 200 行且只含「小智」相关
docker logs xiaozhi-esp32-server --tail 200 2>&1 | grep 小智

# 最近 100 行全部
docker logs xiaozhi-esp32-server --tail 100

# 实时看
docker logs -f xiaozhi-esp32-server 2>&1 | grep --line-buffered 小智
```

复现步骤建议：唤醒 → 立刻追问「北京天气如何」→ 马上执行上述命令，根据是否出现 OTA POST、是否出现 WebSocket 连接/文本消息，按上面表格逐项对照即可明确问题在「未连上自建」还是「已连上但业务/配置问题」。
