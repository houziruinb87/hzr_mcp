# 自建小智服务（xiaozhi-esp32-server）配置清单

依据 [官方仓库](https://github.com/xinnan-tech/xiaozhi-esp32-server/tree/main) 与 [常见问题 FAQ](https://github.com/xinnan-tech/xiaozhi-esp32-server/blob/main/docs/FAQ.md) 整理。**Open-XiaoAI 的 config.py 只改「小智服务器地址」那三行，其余不要动。**

---

## 一、你现在缺什么（一句话）

- **大模型（LLM）**：至少要配一个能用的，例如**智谱 glm-4-flash**（入门全免费，但要**注册并填 API Key**）。
- **语音合成（TTS）**：至少要配一个，例如**灵犀流式 LinkeraiTTS**（入门全免费，一般要注册/配置）。
- **智控台参数**：`server.ota`、`server.websocket`、`server.secret` 填对，且 NAS 上 `data/.config.yaml` 里的 **manager-api.secret** 与智控台一致，Server 才能从智控台拉配置。
- **智能体**：在智控台里建一个智能体，并给它选好 LLM + TTS，设备（Open-XiaoAI）绑到这个智能体上。

下面按步骤做即可。

---

## 二、步骤 1：确认小智服务与智控台已启动

- 在 NAS 上确认 xiaozhi 相关容器都在跑（db、redis、web、server）。
- 浏览器打开智控台：**http://192.168.1.100:18002**（改成你的 NAS IP），能登录即可。

---

## 二、步骤 2：填好 manager-api.secret（Server 才能读智控台配置）

1. 智控台登录 → **参数管理** → 找到 **server.secret**，复制参数值。
2. 在 NAS 上编辑小智目录下的 **data/.config.yaml**，保证有：
   ```yaml
   manager-api:
     url: http://xiaozhi-esp32-server-web:8002/xiaozhi
     secret: 这里粘贴刚才复制的 server.secret
   ```
3. 保存后执行：`docker restart xiaozhi-esp32-server`。  
之后 Server 会从智控台拉配置；若这里没填对，智控台里改的 LLM/TTS 都不会生效。

---

## 三、步骤 3：在智控台设置对外地址（参数管理）

在 **参数管理** 里确认或修改（改成你的 NAS 局域网 IP）：

| 参数编码 | 建议参数值 |
|----------|------------|
| **server.ota** | `http://192.168.1.100:18002/xiaozhi/ota/` |
| **server.websocket** | `ws://192.168.1.100:8000/xiaozhi/v1/` |

保存后若说明里提到需重启设备，再执行一次：`docker restart xiaozhi-esp32-server`。

---

## 四、步骤 4：配置大模型 LLM（必做，你目前没有 API Key 就在这里配）

FAQ 里「入门全免费」推荐用 **智谱 glm-4-flash**，需要先拿 API Key 再在智控台里填。

1. **获取智谱 API Key**
   - 打开：<https://open.bigmodel.cn/> 或 <https://bigmodel.cn/usercenter/proj-mgmt/apikys>
   - 注册/登录 → 在「API 密钥」里**创建密钥** → 复制保存（只显示一次）。
2. **在智控台里配置智谱**
   - 智控台 → **模型配置** → **大语言模型** → 找到或新增「智谱」「ChatGLM」类模型。
   - 填写：
     - **基础 URL**：`https://open.bigmodel.cn/api/paas/v4/`
     - **模型名称**：`glm-4-flash`
     - **API 密钥**：粘贴刚复制的智谱 Key。
   - 保存。
3. 若暂时不想用智谱，可改用其他支持 OpenAI 兼容接口的 LLM（如 Minimax、阿里百炼等），同样在「模型配置 → 大语言模型」里填对应 base_url、模型名、api_key。

**不配 LLM 或 Key 填错，对话会没反应或报错。**

---

## 五、步骤 5：配置语音合成 TTS（必做）

FAQ 里「入门全免费」推荐 **LinkeraiTTS（灵犀流式）**。

1. 智控台 → **模型配置** → **语音合成（TTS）**。
2. 找到或添加 **灵犀流式 / Linkerai** 类 TTS，按界面说明注册/配置（若有 API Key 或应用 ID 则填入）。
3. 保存。

**不配 TTS 或配置错误，小智即使有文字回复也不会出声。**

---

## 六、步骤 6：创建智能体并绑定 LLM + TTS

1. 智控台 → **智能体管理** → 新建或编辑一个智能体（例如「爸爸最帅」）。
2. 在智能体配置里：
   - **大语言模型(LLM)**：选你在步骤 4 里配好的（如智谱 glm-4-flash）。
   - **语音合成(TTS)**：选你在步骤 5 里配好的（如灵犀流式）。
3. 保存配置。  
智控台若提示「保存后需重启设备生效」，则执行：`docker restart xiaozhi-esp32-server`。

---

## 七、步骤 7：设备绑定（验证码）

1. 先把 **Open-XiaoAI 的 config.py 里仅改「小智服务器」那三行**（见下节），指向你的 NAS 小智，然后重启 Open-XiaoAI，让音箱连上。
2. 对音箱做一次唤醒并说一句话，触发一次与小智的对话。
3. 在 NAS 上执行：`docker logs open-xiaoai-xiaozhi`，在输出里找**验证码**（或「控制面板」后的数字）；或智控台「设备管理」里有时会显示待绑定设备/验证码。
4. 智控台 → **设备管理** → **添加设备** → 输入该验证码，完成绑定。

绑定成功后，该设备就会使用你在步骤 6 里选的智能体（含 LLM + TTS）。

---

## 八、Open-XiaoAI 的 config.py：只改这三行

**除下面这一块外，不要改 config.py 里任何其它内容（唤醒词、vad、before_wakeup 等都不动）。**

把「小智服务器」改成你自建服务的地址（NAS 上全模块部署时，OTA 走智控台 web，对话走主服务 8000）：

```python
    "xiaozhi": {
        "OTA_URL": "http://192.168.1.100:18002/xiaozhi/ota/",   # 智控台 web 的 OTA（改为你 NAS IP）
        "WEBSOCKET_URL": "ws://192.168.1.100:8000/xiaozhi/v1/", # 小智主服务 WebSocket（改为你 NAS IP）
        "WEBSOCKET_ACCESS_TOKEN": "",
        "DEVICE_ID": "",
        "VERIFICATION_CODE": "",  # 首次绑定后可从智控台或日志查看
    },
```

若 Open-XiaoAI 和 xiaozhi 在同一台 NAS 且用 Docker 网络，也可用容器名（需 Open-XiaoAI 加入 xiaozhi 所在网络）：

- `OTA_URL`: `http://xiaozhi-esp32-server-web:8002/xiaozhi/ota/`
- `WEBSOCKET_URL`: `ws://xiaozhi-esp32-server:8000/xiaozhi/v1/`

改好后保存，重启 Open-XiaoAI 容器使配置生效。

---

## 九、自查清单（按顺序打勾）

- [ ] 智控台能打开并登录（http://NAS的IP:18002）
- [ ] **参数管理** 里 server.ota、server.websocket 已设为你的 NAS 地址
- [ ] NAS 上 **data/.config.yaml** 里 manager-api.secret 已填且与智控台 server.secret 一致，并已重启 xiaozhi-esp32-server
- [ ] **模型配置** 里已配置至少一个 **大语言模型（LLM）** 并填写有效 API Key（如智谱 glm-4-flash）
- [ ] **模型配置** 里已配置至少一个 **语音合成（TTS）**（如灵犀流式）
- [ ] **智能体管理** 里有一个智能体，且该智能体已选择上述 LLM 和 TTS 并保存
- [ ] **设备管理** 里已用验证码绑定你的设备（Open-XiaoAI 对应设备）
- [ ] Open-XiaoAI 的 **config.py 只改了 xiaozhi 的 OTA_URL、WEBSOCKET_URL**（及必要时 VERIFICATION_CODE），其余未动
- [ ] 已重启 xiaozhi-esp32-server 和 open-xiaoai-xiaozhi 使所有配置生效
