# 极空间安装 XiaoMusic 实操步骤

按 [XiaoMusic 官方 README 快速入门](https://github.com/hanxi/xiaomusic?tab=readme-ov-file#-%E5%BF%AB%E9%80%9F%E5%85%A5%E9%97%A8%E6%8C%87%E5%8D%97) 在极空间上一步步完成安装。完成后访问 **http://极空间IP:58090** 在 Web 里配置小米账号与设备。

---

## 第一步：在极空间里建好两个目录

1. 打开极空间 **文件管理**。
2. 新建两个文件夹（路径按你习惯来，下面用占位符，请改成你实际路径）：
   - **音乐目录**：用来放 mp3/flac/故事等，例如：
     - `docker/xiaomusic/music`  
     - 或你已有的音乐库目录，如 `音乐`
   - **配置目录**：给 XiaoMusic 存配置与数据库，例如：
     - `docker/xiaomusic/conf`
3. 记下这两个目录在极空间里的**完整路径**（后面创建容器时要填）。  
   若你打算用 Docker Compose，路径通常类似：`/volume1/docker/xiaomusic/music` 和 `/volume1/docker/xiaomusic/conf`（极空间实际卷名以你界面为准）。

---

## 第二步：拉取 XiaoMusic 镜像

1. 打开极空间 **Docker** 应用。
2. 左侧选 **镜像** → 选 **仓库** 或 **自定义拉取**：
   - **能访问 Docker Hub**：搜索 `hanxi/xiaomusic` → 选第一个 → **下载**，版本选 **latest**。
   - **国内网络**：点 **自定义拉取**，输入下面其一后点 **拉取**：
     - `docker.hanxi.cc/hanxi/xiaomusic`
     - 或 `m.daocloud.io/docker.io/hanxi/xiaomusic`
3. 等下载完成，在 **本地镜像** 里能看到 `hanxi/xiaomusic` 或 `docker.hanxi.cc/hanxi/xiaomusic`。

---

## 第三步：用镜像创建容器（图形界面方式）

1. 在 **本地镜像** 里**单击选中**刚拉取的 xiaomusic 镜像，点 **添加到容器**（或「创建容器」）。
2. 在创建页面里按下面设置，**不要**多挂载「主题目录」等无关路径：

   **（1）文件夹路径 / 卷**

   | 宿主机路径（极空间）     | 装载路径（容器内） |
   |--------------------------|--------------------|
   | 你的「音乐」目录完整路径 | `/app/music`       |
   | 你的「配置」目录完整路径 | `/app/conf`        |

   **（2）端口**

   | 本地端口 | 容器端口 |
   |----------|----------|
   | `58090`  | `8090`   |

   （若 58090 已被占用，可改成如 `58091`，后面 Web 访问和设置里的端口都改成同一数字。）

   **（3）环境变量**

   - 添加一条：
     - 名称：`XIAOMUSIC_HOSTNAME`
     - 值：**极空间的局域网 IP**（例如 `192.168.1.100`），**不要写端口**。
   - 不要在这里填小米账号、密码（按官方说明在 Web 里填）。

3. 其他保持默认（网络用 **bridge** 即可），点 **应用** / **确定** 创建并**启动**容器。

---

## 第四步：Web 里完成首次配置

1. 在电脑上**关闭代理**，浏览器打开：
   ```text
   http://你的极空间IP:58090
   ```
   例如：`http://192.168.1.100:58090`

2. 打开页面后点 **设置**，带 **\*** 的为必填：
   - **小米账号**：与音箱绑定的账号（是**小米 ID**，不是手机号；在手机 设置 → 个人中心 可查）。
   - **小米密码**：该账号密码。
   - **XIAOMUSIC_HOSTNAME(IP或域名)**：填极空间 IP，**不要带端口**（与上面环境变量一致）。
   - **外网访问端口**：填 `58090`（若你第三步用了其他本地端口，这里填那个端口）。

3. 点 **保存**，稍等后应出现**设备列表**，选择你的 **小米音响 Pro（OH2P）**。

4. 回到**首页**，确认顶部已选好设备，即可：
   - 对小爱说：「播放歌曲 xxx」「播放歌单 某文件夹名」；
   - 或在网页里点播测试。

---

## 第五步：试播与可选设置

- **试播**：在音乐目录里放几首 mp3，对小爱说「播放歌曲 歌名」或「播放歌单 文件夹名」。
- **刷新列表**：新增/删除文件后，可说「小爱同学，刷新列表」，或在设置里开启「文件监控」。
- **续播**：在设置里打开 **「启用继续播放」**，每个歌单会记住上次播到哪一首（详见 [小爱播放NAS音乐-XiaoMusic部署.md](./小爱播放NAS音乐-XiaoMusic部署.md)）。
- **听书/故事**：在 music 下建子文件夹（如 `睡前故事`），把 mp3 放进去，说「播放歌单 睡前故事」；官方推荐听书用 [epub2mp3](https://github.com/hanxi/epub2mp3)。

---

## 若设备列表不出现

- 确认绑定小爱音箱的是**创建者**账号，不是家庭管理员。
- 确认设置里 **XIAOMUSIC_HOSTNAME** 与 **外网访问端口** 和实际访问地址一致（IP 无端口，端口用 58090）。
- 可重启容器再保存一次设置；仍不行可看 [FAQ #99](https://github.com/hanxi/xiaomusic/issues/99)、[极空间安装教程 #297](https://github.com/hanxi/xiaomusic/issues/297)。

---

## 为何「唤醒 XiaoMusic 时小爱同学也会回答」

当前是**不刷机**方案：小爱音箱仍连**小米云**，XiaoMusic 通过「小米账号 + 获取对话/语音口令」在同一台设备上响应「播放歌曲」「播放歌单」。

- 你说「小爱同学，播放歌曲 xxx」时，**同一句语音**会同时被：
  - **小米云**收到 → 小爱同学可能播天气、报时或说一句话；
  - **XiaoMusic** 收到 → 从 NAS 播歌。
- 所以会出现「两边都回答」：小爱插话 + XiaoMusic 播歌。

不刷机的前提下，无法从协议上彻底关掉小米云的应答，只能尽量用固定句式（如严格说「播放歌曲 xxx」）、或接受双响应。

## 用 Open-XiaoAI 实现「播歌时小爱不回答」+ 与 XiaoMusic 联动

若已给音箱刷 **Open-XiaoAI** 固件并让 Client 连到你的 Server（4399），则：

- **麦克风与播放**都走你的 Server，**不再经小米云** → 小爱同学不会收到这台设备的语音，也就**不会插话**。
- 代价是：XiaoMusic 依赖的「小米对话/设备」链路也断了，**不能再用原来的方式**让 XiaoMusic 直接响应这台音箱的语音。

因此「联动」的含义是：在你的 **Open-XiaoAI Server**（如 xiaozhi 或 oxa-server）里，自己处理「播放歌曲/歌单」意图，再决定是调 XiaoMusic 还是直接播 NAS 上的文件。

### 推荐做法（思路）

1. **在 4399 Server 里做意图识别**  
   当识别到用户说「播放歌曲 xxx」「播放歌单 xxx」时，不交给小智/大模型回复，而是走「播歌」分支。

2. **播歌分支二选一（或组合）**  
   - **方案 A**：若 XiaoMusic 提供 HTTP API（如点播、歌单），在你的 Server 里请求极空间上的 XiaoMusic（如 `http://极空间IP:58090/...`），拿到播放 URL 或列表后，用 Open-XiaoAI 的 `start_play` / 音频流下发给音箱播放。  
   - **方案 B**：Server 直接读 NAS 上的音乐目录（SMB/NFS 或挂载到跑 4399 的机器），按「歌名/歌单」匹配文件，生成播放 URL 或本地路径，再通过 Open-XiaoAI 协议把音频推给音箱。

3. **效果**  
   - 点歌、播歌由你的 Server + XiaoMusic（或 NAS 文件）完成；  
   - 语音链路完全在你这边，**小爱同学不会回答**。

### 实施注意

- XiaoMusic 官方是否暴露「点播/歌单」的 HTTP API 需查 [官方文档](https://xdocs.hanxi.cc/) 或 GitHub Issues；若无，就用方案 B（Server 直接读 NAS 音乐目录 + 自己推流）。
- Open-XiaoAI 的播放能力：Client 支持 `start_play` 和接收二进制音频流（如 tag=`play`），Server 侧在 xiaozhi/oxa-server 里调用对应接口即可把 NAS 或 XiaoMusic 的音频推到音箱。

### 小结

| 场景 | 小爱同学会回答吗 | 如何与 XiaoMusic 联动 |
|------|------------------|------------------------|
| 不刷机，仅 XiaoMusic | 会（小米云仍收到语音） | 无需联动，XiaoMusic 通过小米对话响应；可尽量用固定口令减轻双响应 |
| 刷 Open-XiaoAI | **不会**（语音不走小米云） | 在 4399 Server 里识别「播放歌曲/歌单」→ 调 XiaoMusic API 或读 NAS → 用 open-xiaoai 协议推流到音箱 |
